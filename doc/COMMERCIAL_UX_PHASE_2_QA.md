# SmartCinema 商业购票纵向切片 QA

> 日期：2026-07-18
> 分支：`zcjx/smart_cinema`
> 范围：商业路线图 Phase 2、Phase 3 与 Phase 4 的首个可运行闭环
> 状态：通过；刷新恢复、售后与内部工具仍按路线图继续

## 1. 结论

生产入口已经从“影院功能仪表盘”切换为可完成交易的消费者购票漏斗：用户看到明确影片、影院、日期、时间、制式和退改政策，先选票种/票数，再用语义化 DOM 座位图手动选择或请求连座推荐，随后创建真实 10 分钟 `SeatHold`，登录/注册后消费同一 hold，生成订单与电子取票码。

本阶段不是只改 CSS：页面的票数、库存、价格、锁座、身份和订单全部来自 v3 应用服务；UI 不直接访问 LocalStorage。旧 Canvas 功能台保留为 `legacy.html`，仅用于迁移期回退和覆盖对照。

## 2. Before / After / Why

| Before | After | Why |
| --- | --- | --- |
| 首页先让用户选择技术影厅和星期 | 首屏给出电影、影院、日期、具体场次、制式、语言和起价 | 先明确购买对象，降低错误场次风险 |
| 推荐问卷收集姓名、年龄和关系 | 票数确定后提供“帮我选连座”和靠中/靠后/过道/无台阶偏好 | 删除不必要 PII，把推荐输入限制在座位决策事实 |
| Canvas 只有整图语义 | 180 个座位均为可聚焦按钮，提供排/座/附加费/类型/状态 | 支持键盘、读屏、触控和逐座错误定位 |
| 热图、评分、设置、模拟、AI 与订单争夺注意力 | 生产入口只保留场次、票、座位、金额、政策和下一步 | 页面层级服从购买依赖，而不是源码模块清单 |
| 订单按钮位于长页面后方 | 桌面右侧 sticky 摘要，手机底部固定已选/总价/CTA | 选座时持续看见完成度、成本和下一步 |
| 结算意图没有占用库存 | 继续后原子创建 `SeatHold` 并显示 10 分钟倒计时 | 避免“看似进入结算但座位仍可被抢”的伪锁座 |
| 登录发生在价值建立前 | 访客可浏览、推荐和锁座，仅确认订单前登录 | 推迟身份摩擦，并保持同一草稿和 hold |
| 登录弹窗可因背景手势误关 | 有固定关闭键；背景点击和内容拖出均不关闭；叠层只由最上层响应 Esc | 防止输入或文本选择手势丢失表单/锁座上下文 |

## 3. 架构落点

```text
index.html / public/styles/commercial.css
                ↓
        src/commercial.js
                ↓
src/application/commercial/
├── CommercialApplication
├── CommercialAccountService
├── CommercialBookingService
└── RecommendSeatBlock
                ↓
src/domain/catalog + booking + order
                ↑
src/infrastructure/
├── DemoCatalogRepository / DemoInventorySeeder
├── LocalStateRepositoryV3 / migrations
├── SessionGuestOwnerRepository
└── BrowserClock / BrowserIdGenerator / BusinessDate
```

关键边界：

- v1→v2→v3 连续迁移只在组合根执行；
- 价格只由 `PricingQuote` 计算，UI 只格式化 Money；
- 锁座和库存映射在同一 revision 更新中提交；
- 注册/登录复用认证用例，但读写 v3 单一状态源；
- 访客 owner 放在 SessionStorage 基础设施适配器，UI 不读写浏览器存储；
- 订单保留电影、影院、场次、票种、座位、价格与退改政策快照。

## 4. 浏览器证据

### 桌面 1440 × 1000

- 页面级横向溢出：0px；
- 生产 DOM 座位按钮：180；
- 当前演示场次确定性已售座位：约 22%，本次实测 40；
- 推荐双座：F 排 8–9（场次不同可出现另一确定性合法组合）；
- 选满后形成票种、优选座位附加费、服务费和合计；
- 锁座后确认 Dialog 显示影片、日期时间、影厅、座位、总价和倒计时；
- 登录层叠在确认页之上，关闭登录不会关闭确认页。

### 手机 390 × 844

| 指标 | 实测值 | 结果 |
| --- | ---: | --- |
| 页面 `scrollWidth` | 390px | 无页面级横向溢出 |
| 座位滚动容器宽度 | 360px | 适配页面内边距 |
| 座位图内容宽度 | 650px | 只在座位容器内横向浏览 |
| 座位按钮数 | 180 | 不因窄屏裁剪数据 |
| 固定底栏 | 已选 X/Y、总价、CTA | 选座过程持续可见 |

完整移动流程已实测：推荐连座 → 锁定座位 → 注册 → 确认订单 → 取票码 → 我的订单；完成后选择清空，已购座位进入 sold。

## 5. 交互与缺陷回归

- 登录/注册始终有右上角关闭按钮；
- 输入框按下后拖到遮罩释放，Dialog 保持打开；
- 直接点击登录背景，Dialog 保持打开；
- Escape 只关闭最上层 Dialog，底层锁座确认继续存在；
- 关闭确认页会把 hold 标记为 released 并清除整组库存映射；
- 座位图始终只有一个 `tabindex="0"`，方向键移动，Space 选择，重绘后焦点保持；
- 陪同席会联动对应轮椅位，勾选用途确认前 CTA 禁用；
- 原本已经存在的孤座不再错误阻止无关的新选择；
- 用户姓名、订单字段和推荐结果均使用安全文本节点渲染；
- 服务器路径解析忽略 query string，并不把请求 URL 原样反射到 404 HTML。
