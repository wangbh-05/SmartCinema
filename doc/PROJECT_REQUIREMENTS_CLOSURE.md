## 1. 范围结论

SmartCinema 是一个无后端、无运行时依赖的本地影院选座项目。当前版本需要证明的是一条可靠、完整、可恢复的演示闭环：

```text
选择电影/影院/场次
→ 选择票种与同行需求
→ 手动选座或请求推荐
→ 查看座位质量、价格和规则
→ 锁座并登录/注册
→ 演示确认订单
→ 查看订单、取消/退票并释放座位
```

“确认订单”代表本地演示中的同步确认，不代表真实扣款。项目不需要为尚不存在的支付渠道建立支付账本、异步回调、清结算或真实退款状态机。

## 3. 明确不进入本轮的能力

- 真实支付渠道、支付账本、异步支付回调与清结算；
- 真实退款渠道回调；
- 真实二维码签名与验票设备集成；
- 优惠券、积分、会员订阅、礼品卡与卖品；
- 地理定位、城市搜索和真实影院接口；
- 伪 WebSocket、随机抢座演出和消费者 AI 聊天浮窗；
- 与购票任务无关的主题调色盘；
- 完整个人资料中心。

现有文字取票码、演示确认、政策驱动取消和本地库存释放已经构成本地项目所需的订单闭环。

## 4. 设计原则

1. 需求能力可以重新安放，但不能因删除旧组件而静默丢失；
2. 不为满足“模块数量”恢复相互争夺注意力的功能面板；
3. 决策辅助必须出现在用户做决定的位置，且默认不遮挡交易主线；
4. 推荐只采集会改变座位结果的事实，不采集成员姓名等无必要信息；
5. 座位状态、热度和质量均同时使用文字、形状或图例，不只依赖颜色；
6. 高频选座反馈即时、克制，减少动态时不产生位移或缩放；
7. UI 不直接读写 LocalStorage，不自行计算价格、库存或退票资格。
