# SmartCinema 内部工具边界

> 状态：Accepted，2026-07-18。适用于 `zcjx/smart_cinema` 分支。

## 1. 决策

`index.html` 是唯一消费者购票入口，只承载“场次 → 票种/票数 → 座位 → 锁座 → 身份 → 订单”交易漏斗。

`internal.html` 是唯一内部操作入口，使用 v3 事实源并要求管理员会话。旧 `legacy.html` 已在满足退出条件后删除；消费者导航不得暴露任何内部入口。

## 2. 能力归属

| 能力 | 消费者购票页 | 账户 Dialog | 内部操作台 |
| --- | --- | --- | --- |
| 影片/影院/场次/票种/选座 | 保留 | — | 仅迁移对照 |
| 连座推荐 | 保留上下文动作 | — | 保留旧算法诊断 |
| 订单、取票码、退票 | 交易完成页 | 保留 | 仅迁移订单调试 |
| 增强可读性/色觉/减少动态 | 应用结果 | 保留设置 | 不作为产品主题系统 |
| 脱敏诊断/完整恢复备份 | 禁止 | 禁止 | 仅管理员，危险操作确认 |
| 用户、订单与库存观察 | 禁止 | 禁止 | 仅管理员，凭证不进入视图模型 |
| 主题色、实时模拟、购前评分、热力图 | 删除 | 删除 | 明确废弃 |
| 浮动 AI 聊天 | 删除 | 删除 | 明确废弃 |

## 3. 依赖与数据边界

- 消费者 UI 只经过 `application/commercial` 访问目录、价格、库存、锁座、账户和订单，不直接读写 Web Storage。
- 内部操作台通过 `bootstrapInternal.js` 组合 v3 application/storage；不导入消费者 controller，也不与消费者页面共享导航入口。
- v2 备份和 demo 明文凭证只是本地演示资产，不得被描述为可投产的认证或安全存储。

## 4. 退出条件

以下条件全部满足后，才能删除 `legacy.html`、`src/app.js`、旧 Canvas/评分/热图 controller 和对应 CSS/测试：

1. v3 库存运维、用户管理和安全备份已有独立内部工具替代；
2. 任一仍有业务价值的旧推荐规则已迁移或明确废弃；
3. v1/v2 存量数据迁移样本与回滚方案已冻结；
4. 浏览器契约能证明生产页不暴露任何内部入口。

## 5. 本阶段验收

- [x] 生产入口无 Canvas、热力图、评分、备份、模拟、后台和浮动 AI；
- [x] 内部操作台具有可见边界说明和搜索引擎禁止元数据；
- [x] 生产页不导航到内部操作台；
- [x] 用独立 v3 运维工具取代整个 legacy 页面；
- [x] 删除无价值的 v2 UI 与其 CSS。
