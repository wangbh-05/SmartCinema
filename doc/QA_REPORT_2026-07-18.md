## 4. 边界与安全审计

- `src/domain`、`src/application`、`src/shared` 不访问 DOM、Canvas、LocalStorage、SessionStorage、系统时间或随机源；
- `src/app.js`、`src/order.js`、`src/ui` 不直接读写浏览器存储；
- 生产首页与结算页无内联 `style`、`onclick` 或字符串 `innerHTML` 渲染；
- 账户、管理员、推荐、评分、订单、聊天和结算字段通过 `textContent` / DOM 节点构造；
- 安全备份默认不包含 credential；完整备份是独立按钮，明确警告后才能继续，取消不会下载；
- CheckoutIntent 使用 SessionStorage，确认订单与库存由同一次 repository update 提交；
- 订单查询、取消和收据均按稳定 `userId` 作用域授权。

## 5. 交互与动效审计

- Dialog 进入 180ms、Chat 进入 160ms，按钮按下 120ms；
- Hover 仅在 `(hover: hover) and (pointer: fine)` 下启用；
- Canvas 点击反馈 240ms，reduced-motion 时完全跳过弹性动画；
- 显式 smooth scroll 会在 reduced-motion 下改用 `auto`；
- resize 中的 Canvas/Heatmap 重绘按 animation frame 合并；
- AI 回复不再制造 200–600ms 随机等待，避免连续回复乱序；
- 高频键盘、文本编辑和座位导航不添加过渡动画。

## 6. 已知限制

- 认证和数据均在本机浏览器中运行，演示密码不是生产级安全存储；不要使用真实凭据；
- 30 列大厅在 320px 的“适应宽度”视图用于总览，精确手动点选应先使用内置缩放，或使用智能推荐；
- 语音提示取决于浏览器是否提供 SpeechSynthesis；不可用时核心流程不受影响；
- realtime 是可注入、可测试的本地事件模拟器，不是真实 WebSocket 服务；
- 原生文件下载/选择与 destructive confirm 仍依赖浏览器平台 UI。

以上限制均已在 README 或界面文案中明确，不影响当前项目约定范围内的完成标准。
