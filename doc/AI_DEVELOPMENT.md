## 一、产品设计

### 1.1 产品定位

SmartCinema 是一款**智能影院选座系统**，面向影院顾客，帮助用户快速找到最佳观影座位。核心差异化能力是**基于约束规则的智能推荐算法**：用户只需提供人数、年龄、观影类型，系统自动推荐最优座位组并给出评分。

### 1.2 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 个人观影者 | 独自观影 | 快速找到视野最好的单座 |
| 情侣 | 2人 | 中间区域连续双座，兼顾私密性 |
| 亲子家庭 | 2人（1大1小） | 避开前三排保护少年视力 |
| 多口家庭 | 3-5人 | 中后排连续座位，方便照顾家人 |
| 朋友/团体 | 2-20人 | 同排连座，成员不分散 |
| 老年观众 | 60岁以上 | 避开后三排方便出入 |

### 1.3 核心功能模块

1. **Canvas 弧形影院引擎**：纯 Canvas 2D API 绘制弧形座位图，三放映厅切换
2. **智能推荐引擎**：硬约束规则 + 评分排序算法
3. **影院热度地图**：以座位边框颜色常驻显示，并提供独立热度图 Canvas，一周内动态切换
4. **观影体验评分**：四维度系统评分 + 观众手动评分综合
5. **无障碍支持**：语音提示、大字体、高对比度、色盲友好模式
6. **订单中心**：独立确认页 + 历史订单 + 取消/退票 + LocalStorage 持久化

## 二、用户分析

### 2.1 用户痛点

1. **选择困难**：面对数百个座位不知道选哪个最好
2. **信息不对称**：不知道哪个区域观影体验最佳
3. **群体协调难**：多人观影时难以找到连座
4. **特殊需求被忽视**：带小孩/老人有特殊座位需求
5. **价格不透明**：不同区域的定价差异不直观

### 2.2 解决方案映射

| 痛点 | 解决方案 |
|------|---------|
| 选择困难 | 智能推荐一键选座 + 四维度评分量化体验 |
| 信息不对称 | 热度地图直观展示座位受欢迎程度 |
| 群体协调难 | 同排连续搜索算法 + 团体约束规则 |
| 特殊需求 | 少年/老年人硬约束：禁止特定排 |
| 价格不透明 | 基于真实影院定价模型的分区价格 + 热度与价格正相关 |

### 2.3 用户旅程

```
进入页面 → 登录(或注册) → 选择放映厅/日期
    → 选择选座方式:
        A. 智能推荐: 填表单 → 一键获得推荐 → 应用
        B. 手动选座: 点击/拖拽 → 实时看评分
    → 查看体验评分 → 提交订单 → 确认支付 → 完成
```

## 三、AI 生成内容说明

### 3.1 AI 参与范围

本项目与 AI（Claude Code）协同开发。AI 参与了以下工作：

| 类别 | 内容 | AI 参与度 |
|------|------|----------|
| 架构设计 | 模块划分、数据流、组件树 | 30%（人工主导，AI建议） |
| Canvas 引擎 | 弧形算法、绘制逻辑、事件系统 | 60%（AI编写主体，人工调试修正） |
| 推荐算法 | 约束规则、评分函数、排序策略 | 50%（规则由需求文档确定，AI实现+优化） |
| 评分引擎 | 维度设计、权重、计算公式 | 40%（人工定权重，AI实现计算逻辑） |
| 订单系统 | CRUD、状态机、数据持久化 | 70%（标准逻辑，AI高效实现） |
| CSS 样式 | 布局、主题、无障碍样式 | 60%（AI生成，人工调整） |
| 文档 | 设计文档、流程图、本说明书 | 80%（AI撰写，人工审核修正） |
| Bug 修复 | 排查、定位、修复 | 90%（AI诊断+修复，人工验证） |

### 3.2 关键 AI 生成代码段

- **`src/core/Cinema.js`**：Canvas 弧形布局、座位绘制、银幕光锥、热度边框、拖拽框选、键盘选座、tooltip。AI 经历 v4→v5→v6 多次迭代重写
- **`src/modules/RecommendEngine.js`**（305行）：约束规则定义、四种推荐策略、评分排序、理由生成
- **`src/modules/ScoreEngine.js`**（320行）：四维度评分计算、等级映射、详情生成
- **`order.html`**（320行）：独立订单确认页面，sessionStorage 跨页数据传递

### 3.3 AI 协作模式

本项目采用 **"需求驱动 + 迭代修正"** 的 AI 协作模式：

1. **需求输入**：将大作业要求文档逐条翻译为技术规格
2. **AI 生成**：AI 编写初始代码
3. **人工测试**：在浏览器中运行验证
4. **反馈迭代**：发现问题 → 描述给 AI → AI 修复 → 再测试
5. **文档同步**：每次重大变更后更新 REQUIREMENTS_CHECK.md 和 ROADMAP.md

## 四、学生修改内容及思路

### 4.1 人工主导的设计决策

| 决策 | 理由 | 提出者 |
|------|------|--------|
| Canvas 白底 + 页面深色 | 热度边框在白色上可见，深色背景模拟影院 | 学生 |
| 热度边框 + 独立热度图双呈现 | 选座时可直接看边框热度，独立图满足热度地图验收 | 学生/AI协作 |
| 级联推荐表单（人数→年龄→类型） | 模拟真实购票流程，降低认知负荷 | 学生 |
| 独立订单确认页 | 多一步确认减少误操作，符合电商惯例 | 学生 |
| 真实影院定价模型驱动热度 | 让热度有实际意义而非随机数 | 学生 |

### 4.2 主要人工修正

1. **评分计算 bug**：AI 生成的 `Math.round(score * 100)` 误将 0-10 分映射为 0-1000，学生发现并修正为 `* 10`
2. **Canvas 初始渲染 bug**：切换放映厅后 Canvas 不显示，学生诊断为 `switchHall()` 缺少 `redraw()` 调用
3. **热度显示迭代**：从独立 Canvas 热力图 → 嵌入座位图的光晕 → 边框颜色，经三轮迭代达到最终方案
4. **配色调整**：深色→白底主题、已售灰+热度边框、色盲友好配色，多轮微调

### 4.3 功能增强（学生自主添加）

- 拖拽框选（原为加分项，学生自主实现）
- 语音提示联动切换厅/日期
- 色盲模式 Canvas + CSS 双轨切换
- 一周热度动态切换（周末上座率加成）
- 订单确认、取消、退票闭环与已售座位持久化
- Canvas 键盘选座（方向键移动，Enter/Space 选择）

## 五、用户体验优化过程

### 5.1 优化历程

| 版本 | 主要优化 | 用户反馈驱动 |
|------|---------|-------------|
| v1 | 基础骨架：Canvas + 三个厅 + 登录 | 初始版本 |
| v2 | UI 重构：白底Canvas、热度边框、级联表单、独立订单页 | 深色主题与 Canvas 白色间隔不协调；热度投影效果差 |
| v3 | 功能补全 + Bug 修复：姓名输入、热度周切换、手动评分、色盲模式 | 推荐多年龄段禁排失效；friends/parent_child 类型遗漏 |

### 5.2 关键 UX 改进

1. **选座交互简化**：从 Ctrl+点击多选 → 独立切换（单击选中/取消），降低操作门槛
2. **座位号移除**：行列标签在坐标轴标明，座位内不再显示"4-5"，画面更干净
3. **银幕光锥**：光形跟随曲线而非矩形，视觉更真实
4. **评分布局**：从冗长的多行堆积 → 总分一行突出 + 四维度紧凑排列
5. **订单流程**：从侧边栏直接创建 → 独立确认页，减少误操作

## 六、设计决策与思路

### 6.1 技术选型

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 图表技术 | 纯 Canvas 2D API | 大作业要求"未使用第三方 Chart 库" |
| 模块系统 | ES Modules | 现代浏览器原生支持，无需打包工具 |
| 数据存储 | LocalStorage | 无后端依赖，纯前端项目 |
| 跨页通信 | sessionStorage | 订单摘要仅需一次传递，无需持久化 |
| CSS 架构 | 分层 CSS 文件 | reset → variables → layout → components → responsive → accessibility |

### 6.2 架构决策

1. **SeatData 作为单一数据源**：所有模块（Cinema、RecommendEngine、ScoreEngine、OrderManager）共享同一个 SeatData 实例
2. **事件驱动 UI 更新**：Canvas 通过 `CustomEvent('selectionChange')` 通知 app.js 更新统计和评分
3. **热度算法核心思想**：`dist = sqrt(rowDist² × 0.4 + colDist² × 0.6)`，列权重 > 行权重，模拟真实影院"中间列比中间排更贵"的规律
4. **推荐评分优先于约束**：先过滤禁排 → 找连续座位 → 评分排序 → 返回最优，确保推荐结果在合规前提下体验最佳

### 6.3 配色决策

- **Canvas 白底 vs 页面深色**：解耦两套配色体系，Canvas 内 `CLR` 常量独立管理
- **色盲友好配色**：提供可切换的 `CLR_CB` + `HEAT_CB`，使用蓝/橙替代红/绿
- **热度三色梯度**：从 `<0.3 冷蓝` → `0.3-0.6 暖琥珀` → `>0.6 热红`，阈值可调
