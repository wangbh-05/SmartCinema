


agent.md：SmartCinema 开发约束与技术栈文档

## 1. 项目概览

本项目为《Web前端技术实训》2026夏清华大学软件学院大作业，旨在开发一个基于 Web 的”智能影院选座系统”。项目核心在于使用原生技术栈实现复杂的 UI 交互与数据逻辑，并体现 AI 协同开发的思考过程。

> 必须遵循”别让用户思考（Don't Make Me Think）”原则，通过系统主动推荐座位和优化操作流程，让用户快速完成选座与购票。

## 2. 技术栈约束 (必须严格遵守)

* **基础技术：** 原生 HTML5, CSS3, JavaScript (ES6+)。
* **绘图技术：** **必须使用 Canvas API** 绘制电影院布局及影院热度地图。
* **严禁使用**任何第三方图表库（如 ECharts, D3, AntV 等）。
* **数据存储：** 仅限使用 `LocalStorage` 或本地 `.json` 文件（无后端数据库依赖）。
* **响应式适配：** 必须支持 PC 端、iPad 端、手机端浏览。
* 实时交互：可使用 `WebSocket` 模拟多人在线选座（加分项）。

## 3. 影院规格

| 放映厅 | 座位总数 | 排数 | 每排座位数 | 座位排列 |
|--------|---------|------|-----------|---------|
| 小厅   | 100座   | 10排 | 每排10座  | 弧形排列 |
| 中厅   | 200座   | 10排 | 每排20座  | 弧形排列 |
| 大厅   | 300座   | 10排 | 每排30座  | 弧形排列 |

* 每个座位需标明座位号（排号 + 座号）
* 座位必须**弧形排列**（因影院放映厅是弧形结构）
* **座位颜色规范**：空座=绿色、选中未售=黄色、已售=红色

## 4. 核心功能需求

### 4.1 基础模块（30分）

* **登录注册（10分）：**
  * 用户登录注册，普通用户注册获得会员资格
  * 登录认证后方可操作
  * 管理员用户登录可进入后台管理操作

* **主界面座位图（20分）：**
  * 使用 Canvas 技术绘制三个放映厅的座位图
  * 未使用任何第三方 Chart 库
  * 座位弧形排列，标明排号和座号
  * 颜色标识：空座绿色、选中未售黄色、已售红色

### 4.2 功能模块（60分）

* **模块1 - 智能推荐选座（10分）：**
  * 观众分类：成年人、老年人、少年
  * 选座类型：个人票、情侣票、家庭票、团体票
  * **排座规则**（必须严格遵守）：
    * 少年（15岁以下）：不能坐前三排
    * 老年人（60岁以上）：不能坐最后3排
    * 情侣：优先推荐中间区域连续双座
    * 家庭：优先推荐中后排连续座位
    * 团体票（5-20人）：成员不能分开，必须坐同一排；团体中有老年人或少年要遵循上述规则
    * 非以上类别成年人可以随意坐

* **模块2 - 手动选座模式（5分，拖拽+5分）：**
  * 单选
  * 多选（Ctrl+鼠标单击座位）
  * 支持与推荐座位交互（推荐不满意再手动选择其他座位）
  * 拖拽选座（加分项）

* **模块3 - 影院热度地图（10分）：**
  * 使用 Canvas 绘制观众热度分布
  * 色彩标识：**红色=热门区域，黄色=一般区域，蓝色=冷门区域**
  * 动态显示**一周内**的厅内热度区域变化
  * 通过算法计算座位热门程度

* **模块4 - 观影体验评分（10分）：**
  * 评分指标：视角、与银幕距离、周围空位情况
  * 评分结果：极佳、优秀、一般
  * 观众观影后手动评分，与系统评分综合计算
  * 提供推荐理由帮助用户理解选择逻辑

* **模块5 - 无障碍模式（10分）：**
  * 大字体模式
  * 高对比度模式
  * 色盲友好模式
  * 可选语音提示（帮助老年/残障人士使用）

* **模块6 - 订单中心（10分）：**
  * 预订、取消预订、购票、退票
  * 所有数据存储在 LocalStorage 或本地 JSON 文件
  * 无后端数据库依赖

### 4.3 交互与视觉设计（5分）

* **交互要求：** 提交用户操作流程图，操作流程尽量简化（≤3步），界面交互符合用户逻辑与思维习惯
* **视觉风格：** 强调科技感与美学，提交设计文档（色彩方案、布局设计、字体选择、设计风格说明）
* 可参考网站：Apple, Tesla, OpenAI, Netflix, 猫眼电影, 淘票票

### 4.4 说明文档（5分）

* 产品设计过程说明
* 用户分析过程说明
* AI 生成内容说明
* 学生修改内容及思路说明
* 用户体验优化过程记录
* 设计决策和思路说明
* 列出所有参考资料

### 4.5 额外加分项（10分）

* **AI 观影问答式顾问（3分）：** 根据用户输入自动推荐最佳座位，提供推荐理由（视角、噪音、便捷性）
* **拖拽选座、动画交互优化（2分）**
* **WebSocket 实时座位更新（多人在线模拟）（3分）**
* **自设计影院布局或个性化界面（2分）**

## 5. 目标用户群体

| 用户类型 | 需求特征 |
|---------|---------|
| 大学生 | 多人同行，预算有限，快速选座 |
| 情侣 | 偏爱中后排，优先双人连续座位 |
| 家庭 | 多人观影，连续座位需求，照顾儿童和老人 |
| 老年人 | 操作能力弱，字体大，界面简洁 |
| 团体观众 | 5~20人，要求同排连续座位 |

|-------|------|

## 7. 提交要求

* **提交格式：** zip 压缩包，文件名为”组名_大作业1”
* **源代码：** 整理为 1 个 html 文件，命名为 `index.html`，可直接用浏览器打开
* **说明文档：** 转成 PDF，命名为 `report.pdf`，需写明姓名、学号和联系邮箱
* **迟交规则：** 每迟交一天扣除 10% 的分数

## 8. AI 协同开发与文档要求

本项目强制要求 AI 辅助开发，但必须保留独立思考过程。

### 8.1 AI 使用指南

* 支持使用 Cursor, Trae, ChatGPT, Claude 等工具辅助编码与逻辑设计

### 8.2 必须提交的《AI 开发说明书》内容

在项目结束时，必须汇总以下内容，用以证明开发过程的独立性与深度：

1. **产品设计过程：** 用户分析、核心功能决策说明
2. **AI 协同记录：** 记录 AI 生成的内容、学生对代码的优化与修正过程
3. **设计决策说明：** 重点解释以下”为什么”问题：
   * 为什么推荐这样的座位？
   * 为什么设计这样的交互流程？
   * 为什么采取当前的界面布局？
   * 为什么要使用该系统（用户价值分析）

## 9. 开发路径建议

1. **Step 1：原型构建。** 使用 Canvas 搭建基础影院网格，实现基本的响应式布局与本地数据存储连接。
2. **Step 2：核心算法。** 编写推荐算法，根据输入规则计算并高亮推荐座位。
3. **Step 3：交互增强。** 添加拖拽、点击选中与无障碍辅助逻辑。
4. **Step 4：评分引擎。** 编写观影体验评分函数，将计算结果实时映射到 UI。
5. **Step 5：文档沉淀。** 在开发过程中，请务必记录关键代码的修改版本（Prompt 历史），以便后期撰写《AI 开发说明书》。
