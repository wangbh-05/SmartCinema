# TECH_STACK.md：原生前端架构契约

> 本文档规定了 SmartCinema 系统的核心技术栈。所有 AI 辅助开发必须严格遵循以下原生技术选型，严禁引入未经许可的第三方框架或库。

## 1. 核心技术选型

| Area               | 选型                             | 理由                                       |
| ------------------ | -------------------------------- | ------------------------------------------ |
| **语言**     | Vanilla JavaScript (ES6+)        | 满足实训课程原生开发要求，无框架负担       |
| **构建工具** | 无 (或轻量级静态服务器)          | 直接运行在浏览器环境，体现原生加载机制     |
| **视图渲染** | HTML5 + CSS3 (Flex/Grid)         | 实现高度定制化的响应式布局                 |
| **核心绘图** | **Canvas API (手写)**      | 满足核心评分占比，实现影院布局与热度地图   |
| **状态管理** | 闭包 / 模块级变量 / LocalStorage | 轻量化处理本地数据持久化与选座状态         |
| **模块化**   | ES Modules (import/export)       | 保持代码结构清晰，无需打包工具即可组织逻辑 |
| **无障碍**   | WAI-ARIA 原生标准                | 确保高对比度与键盘访问支持                 |

## 3. 架构与编码规范

* **Canvas 封装原则**：所有绘图逻辑必须封装为类（Class），严禁在业务逻辑中直接硬编码 `ctx` 方法。必须实现坐标映射与事件委托处理（将点击事件映射至 Canvas 内部座标）。
* **职责分离**：
  * `Data Layer`: 负责数据处理、LocalStorage 读写。
  * `Render Layer`: 纯粹的 Canvas 绘图与 DOM 操作。
  * `Logic Layer`: 选座算法、评分引擎、事件总线。
* **零依赖原则**：除特殊需求外，禁止引入任何第三方库。如需日期、颜色处理，请手动实现轻量级工具函数。

## 4. 开发约束

* **禁止使用框架**：严禁使用 React, Vue, Angular 等框架，严禁使用 Vite, Webpack 等构建工具作为项目依赖，保持项目在浏览器原生环境下的可移植性。
* **响应式策略**：优先采用 CSS Media Queries 实现自适应；若 Canvas 需根据视口缩放，必须实现 `resize` 事件监听器及重绘算法。
* **数据结构规范**：影院座位数据结构应统一为 JSON 格式，便于通过 `JSON.stringify()` 和 `JSON.parse()` 与 `LocalStorage` 交互。

## 5. 变更控制

* 本技术栈为课程大作业的基础规范，如需引入新的扩展库，必须在 PR 中明确说明其实现功能及必要性，并经过验证。
* 任何代码变更必须保证原生环境的兼容性（ES6+ 特性在主流现代浏览器下的支持）。
